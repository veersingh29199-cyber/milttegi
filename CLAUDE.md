# 운행일지·작전판 — 개인용 대리운전 기록 앱 개발 지시서

이 문서는 발주자(대리기사)와 사전에 확정한 기획 전체다. 이 문서 밖의 기능을 임의로 추가하지 말 것.

## 0. 진행 원칙

- 시니어 프런트엔드 개발자 역할. 발주자는 초보이므로 모든 응답·설명·코드 주석은 한국어로, 기술 개념은 초보자가 이해할 비유로 설명한다.
- 진행 순서는 항상 [기획 확인] → [구현] → [테스트]. 작업 시작 전 이해한 목표를 한 문장으로 요약해 확인받는다.
- "7. 구현 순서"의 한 단계가 끝날 때마다 멈춘다: 만든 것 요약 → 실행·확인 방법 안내 → 승인 후 다음 단계.
- 터미널 명령은 실행 전에 무슨 명령이고 왜 필요한지 한 줄로 설명한다.
- 에러 시: 원인 분석 → 해결책 3가지 → 가장 효율적인 것 선택 실행.
- Clean Code 원칙. 모든 함수 위에 역할 설명 한국어 주석. 커밋 메시지 한국어, 단계별 커밋.

## 1. 배경

- 발주자는 부산에서 부업으로 대리운전(하루 3~5콜).
- 평일: 강서구 신호동·명지동 일대 '밀떼기'(자차를 구역에 두고 단거리 콜 연속 수행, 회전수로 버는 전략). 자차 기동력으로 오지 픽업·기피 도착지 콜을 잡는 게 경쟁력.
- 주말: 동래구 일대.
- 목적: 본인 전용 운행 기록 + 데이터 기반 대기 전략 도구. 타인 배포용 아님.
- 핵심 철학: 기능 최소, 입력 마찰 최소(콜 1건 30초), 서버 없음.

## 2. 기술 스택과 절대 제약

- Vite + React + TypeScript + Tailwind CSS.
- 모바일 우선(안드로이드 크롬), 다크 모드 기본, 한 손 조작 최적화, 모던 미니멀 UI.
- PWA(vite-plugin-pwa): 홈 화면 추가, 오프라인 완전 동작.
- 저장은 localStorage만. 서버·DB·로그인·회원가입·계정 금지.
- 외부 API 호출 금지(지도·날씨·경로 포함). 지역 데이터는 정적 JSON 내장.
- 대리운전 플랫폼 앱 화면·알림 읽기/자동화 절대 금지. 데이터는 오직 수동 입력.
- 민감정보는 원칙적으로 없어야 하며, 생기면 `.env`로만 관리·하드코딩 금지.
- 배포 타깃: Vercel.

## 3. 데이터 설계 (localStorage)

파생값(실수령, 회전 간격, 시급)은 저장하지 않고 항상 계산한다 — 수수료율을 나중에 바꿔도 과거 통계가 함께 재계산되도록.

- `dv.schemaVersion`: 숫자, 현재 1. 앱 시작 시 저장 버전과 코드 버전을 비교해 마이그레이션을 실행하는 뼈대를 둔다(지금은 변환 로직 없이 구조만).
- `dv.settings`: `Settings`
  - `platforms`: `{id, name, feeRate}[]` — 초기값 카카오 0.20 / 티맵 0.20. 실제 수수료율을 단정하지 말 것(수시 변동). 설정에 "내 값으로 수정하세요" 배지.
  - `minHourly`: 최소 시급. 초기 15000, 수정 요망 배지.
  - `favorites`: 즐겨찾기 시군구 코드.
  - `customZones`: `{id, parentCode, name}[]` — 내 구역 커스텀 라벨.
  - `sessionGapMin`: 세션 컷 기준(분). 기본 120, 설정에서는 숨김(상수 취급).
- `dv.trips`: `Trip[]` — `{id, at(운행 종료 ISO), platformId, from, to, fromZone?, toZone?, fare, rain, event, memo?}`
- `dv.dailyExpenses`: `{ "YYYY-MM-DD": number }` — 영업일(06시 경계) 기준 날짜별 경비.

## 4. 지역 데이터 (regions.json — 내장)

- 범위: 부산(구·군 16) + 울산(구·군 5) + 경남(시·군 18) = 39개 시군구.
- 행정표준코드(시군구 5자리)를 키로 사용. 동명 지역 구분 + 향후 전국 확장 호환. 개수(16/5/18) 자체 검증.
- 내 구역 프리셋: 강서구 하위 `신호단지`, `명지 중심`, `명지 외곽` 3개(설정에서 추가·수정·삭제 가능). 라벨에 성격(중심/외곽)이 드러나면 기피 프리미엄 분석 재료가 된다.
- 즐겨찾기 초기값: 부산 강서구·동래구·사상구·사하구·북구 / 경남 김해시·창원시.

## 5. 화면 명세 — 하단 탭 3개: [기록] [작전판] [설정]

- **5-1 기록 탭(기본)**: 한 손 30초. ① 현재 시각 자동(탭해 수정) → ② 플랫폼 칩(마지막 선택 기억) → ③ 출발지 → ④ 도착지(각각 내 구역 칩 + 즐겨찾기 칩 한 줄 먼저, '전체' 누르면 시·도→시·군·구 2단 바텀시트, 최근 사용 지역 자동 승격) → ⑤ 요금 숫자패드(+5천/+1만) → ⑥ 비·행사일 토글 → ⑦ 저장(진동/토스트). 당일 목록 스와이프 수정·삭제.
- **5-2 몰아입력 모드**: 놓친 날을 위해 플랫폼 운행내역 보며 옮겨 적기. 날짜 고정, 저장 즉시 다음 건, 직전 도착지를 다음 출발지로 자동 제안. 마지막에 '오늘 경비' 한 칸 → `dv.dailyExpenses`.
- **5-3 작전판 탭**: 요일 자동 모드(월~금 강서 / 토·일 동래, 출발 시군구 필터로 동작, 수동 전환 가능). 잠금(전체 10건 미만). 적응형 해상도(적을 땐 시간대 3구간×주중/주말 굵은 표, 칸당 5건 이상이면 1시간×요일 세분화, 3건 미만 칸은 회색·건수 표시). 위젯: ① 시간대별 회전 간격·시간당 실수령 ② 플랫폼 비교 ③ 도착지 랭킹(회전 빠른 곳/느린 곳+막콜 비율) ④ 주간 요약. 전용 리포트 3종: ⓐ 평일 강서 vs 주말 동래 시간당 실수령 ⓑ 기피 프리미엄(내 구역 중심/외곽) ⓒ 구역 이탈 손익분기.
- **5-4 설정 탭**: 플랫폼 수수료율(수정 요망 배지), 최소 시급(수정 요망 배지), 즐겨찾기·내 구역 관리, 데이터 내보내기/가져오기(JSON), 전체 삭제(2단계 확인), 앱 정보.

## 6. 계산 규칙 (단위 테스트로 고정)

- 실수령 = 요금 × (1 − 플랫폼 수수료율).
- 회전 간격 = 다음 기록 `at` − 이번 기록 `at`(at=운행 종료 시각).
- 세션 컷: 회전 간격 > `sessionGapMin`(120) → 근무 종료. 직전 기록은 막콜. 막콜은 회전 통계 제외, 도착지별 막콜 비율은 별도 집계.
- 영업일 경계 = 오전 6시: 새벽 2시 기록은 전날 영업일·전날 요일로 귀속.
- 세션 시간 = 세션 마지막 `at` − 첫 `at`. 1건짜리 세션은 시급 계산 제외.
- 일 순수익 = 그 영업일 실수령 합 − 그 영업일 경비.

## 7. 구현 순서 (단계마다 멈추고 승인 대기)

1. 프로젝트 셋업 + regions.json + 저장 계층(스키마·마이그레이션 뼈대) + 계산 유틸(순수 함수 분리)
2. 계산 유틸 단위 테스트(Vitest): 자정 넘김, 06시 경계, 막콜·세션 컷, 수수료, 내보내기/가져오기 왕복 직렬화
3. 기록 탭
4. 작전판 탭(적응형 해상도 → 위젯 → 전용 리포트)
5. 설정 탭 + JSON 내보내기/가져오기
6. 몰아입력 모드 + 오늘 경비
7. PWA 설정 + 프로덕션 빌드 + Vercel 배포 안내

## 8. 완성 판정 기준

1. 한 손 30초 안에 1건 기록 완료. 2. 10건 이상이면 작전판이 "지금 이 시간대, 어느 구역" 답 제시. 3. 내보내기→가져오기 왕복 후 데이터 완전 동일. 4. 6장 계산 규칙 단위 테스트 전부 통과. 5. 저사양 안드로이드 크롬에서 버벅임 없음.
